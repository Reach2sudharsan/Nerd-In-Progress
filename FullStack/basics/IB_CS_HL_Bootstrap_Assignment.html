History

Tim Berners-Lee invented the World Wide Web in 1989 while working at CERN. He also developed the foundations for the Web: HTTP, HTML, the WorldWideWeb browser, a server, and the first website for documentation.

Evolution

The different versions of the Web overtime

The web underwent several changes as time passed, primarily security and privacy updates. Especially because people have so much more and easier access to data nowadays, many try and take advantage of that to steal data and engage in other criminal activities.

Future

The web will definitely last very long, and it will evolve substantially in the coming decades. The web will soon be characterized by devleopment technologies such as Quantum Computing, Accelerated Mobile Pages, Progressive Web Apps, Artificial Intelligence, Voice Search, Motion UI, and many more.


Search Engines

Example search engines

Search engines are software systems that are designed to carry out web searches. They search through the World Wide Web in a systematic way for specific information noted in a textual web search query. The search results are usually seen through the search engine results pages, which show a line of different results.

Search Algorithms

A graph of nodes that can be used to represent interconnected web pages (important for search algorithms)

Search algorithms in computer science are designed to search for a particular element from some kind of data structure. In the context of web science, search algorithms will search using different aspects of web pages such as the time that a page has existed and the frequency of the search keywords on the page. PageRank and HITS are two very popular search algorithms. PageRank works by counting the number and quality of inlinks of a page to determine a rough estimate of how important the website is. For HITS, based on the idea that keywords are not everything that matters; there are sites that might be more relevant even if they don’t contain the most keywords. It introduces the idea of different types of pages, authorities and hubs. Search algorithms use complex graph theory for searching; all web pages are essentially nodes of a graph.

Web Crawlers

Purposes:

  • Browses the web in a methodical and automated manner.
  • For each page it finds, a copy is downloaded and indexed. In this process it extracts all links from the given page and then repeats the same process for all found links.
Advantages:
  • Can extract data automatically from various competitors' websites without any hassle.
  • The crawl system should make efficient use of various system resources including processor, storage and network bandwidth.
Disadvantages:
  • They might look at meta data contained in the head of web pages, but this depends on the crawler
  • A crawler might not be able to read pages with dynamic content as they are very simple programs
When Used:
  • Used to traverse the web by downloading documents and following links from page to page.
  • It is a tool for the search engines and other information seekers to gather data for indexing and to enable them to keep their databases up to date.

Parallel Web Crawlers

Purposes:

  • Browses the web in a methodical and automated manner.
  • For each page it finds, a copy is downloaded and indexed. In this process it extracts all links from the given page and then repeats the same process for all found links.
Advantages:
  • Scalability: as the web grows a single process can not handle everything Multithreaded processing can solve the problem
  • Network load dispersion: as the web is geographically dispersed, dispersing crawlers disperses the network load
  • Network load reduction
Disadvantages:
  • Overlapping: parallel web crawlers might index the same page multiple times
  • Quality: If a crawler wants to download ‘important’ pages first, this might not work in a parallel process
  • Communication bandwidth: parallel crawlers need to communicate for the former reasons, which for many processes might take significant communication bandwidth
  • If parallel crawlers request the same page frequently over a short time it will overload servers
When Used:
  • Runs multiple processes in parallel.
  • Used to maximize the download rate while minimizing the overhead from parallelization and to avoid repeated downloads of the same page.
  • Identifying hubs and authorities in the web for the HITS algorithm.

Meta Tags

Purposes:

  • Meta tags are essentially little content descriptors that help tell search engines what a web page is about.
  • The title tag, not strictly a meta-tag, is what is shown in the results, through the indexer
Advantages:
  • A compelling meta description has the power to raise the click-through rate of your organic search results.
Disadvantages:
  • While meta-tags used to play a role in ranking, this has been overused by many pages and therefore meta-tags are not considered by most search engines anymore.
  • A crawler might not be able to read pages with dynamic content as they are very simple programs
When Used:
  • Used to describe the content on a webpage.
  • helps search engines understand what it's about so you can show up in the relevant search results.

Robots.txt

Example search engines

Purposes:

  • A robots. txt file tells search engine crawlers which URLs the crawler can access on your site.
  • A robots.txt file contains instructions for bots that tell them which webpages they can and cannot access.
Advantages:
  • Index certain areas of the website or a whole website.
  • Index specific files on a website (images, videos, PDFs)
  • Prevent duplicate content pages from appearing in SERPs.
  • Index entire sections of a private website (for instance, your staging site or an employee site)
Disadvantages:
  • Robots.txt directives may not be supported by all search engines.
  • Different crawlers interpret syntax differently. Although respectable web crawlers follow the directives in a robots.
When Used:
  • Used mainly to avoid overloading your site with requests
  • Used to identify restricted or private areas of a site's contents.

Web Indexing

Purposes:

  • Search engines index websites in order to respond to search queries with relevant information as quick as possible.
  • Indexing has the additional purpose of giving a page a certain weight, as described in the search algorithms. This way search results can be ranked, after being indexed.
Advantages:
  • Proper website indexing allows search engines to see all of your important pages and gives your site a major boost.
  • Ensure that filed papers are located easily and quickly whenever they are needed.
Disadvantages:
  • Since Indexes physically take up space on the Disk, using the Index will increase the extra disk cost unless necessary.
  • In general, indexes improve performance in our Select queries and slow down DML (insert, update, delete) operations.
When Used:
  • A search index is a body of structured data that a search engine refers to when looking for results that are relevant to a specific query.
  • Detecting web pages of similar topics through graph-theoretical properties only.
